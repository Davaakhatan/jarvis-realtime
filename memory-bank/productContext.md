## Product Context

### Problem Statement
Frontline workers face the challenge of accessing cross-team information accurately and instantaneously. In high-pressure situations, latency or inaccuracies in data can lead to severe consequences. Current solutions fail to meet the demands for real-time, reliable communication tools. Jarvis addresses this gap by providing an intuitive software+hardware solution that supports seamless, intelligent communication and task handling.

### Target Users & Personas

#### 1. Frontline Workers
- **Primary need**: Immediate access to accurate information for real-time decisions
- **Pain points**: Current tools are too slow, require too many clicks, or provide unreliable information
- **Key requirement**: Hands-free, voice-first interaction with interruption-friendly responses

#### 2. Team Leaders
- **Primary need**: Efficient cross-team communication and task management tools
- **Pain points**: Information silos, delayed updates, difficulty coordinating across teams
- **Key requirement**: Seamless information sharing and confirmation mechanisms

#### 3. IT Managers
- **Primary need**: Reliable, secure solution that integrates easily with existing systems
- **Pain points**: Security vulnerabilities, compliance concerns, integration complexity
- **Key requirement**: End-to-end encryption, audit logging, GDPR compliance

### User Stories

| As a... | I want to... | So that... |
|---------|--------------|------------|
| Frontline worker | Receive immediate answers to my queries | I can make decisions quickly and accurately |
| Frontline worker | Interrupt the assistant mid-response | I can redirect when priorities change |
| Team leader | Ensure my team can access and share information seamlessly | We maintain operational efficiency |
| Team leader | Get summarized updates from Jarvis | I can keep the team aligned without manual aggregation |
| IT manager | Have a reliable and secure solution | It integrates easily with our current systems |
| IT manager | Access comprehensive audit logs | I can ensure compliance and troubleshoot issues |

### Value Proposition
Jarvis blends always-on voice interfaces with verifiable LLM reasoning, API data, and GitHub knowledge to minimize latency and avoid hallucinations. Key differentiators:

1. **Zero-hallucination policy**: Every answer must cite verified, authoritative sources
2. **Sub-500ms responses**: Streaming architecture ensures real-time feel
3. **Context persistence**: Maintains conversation history for coherent multi-turn interactions
4. **API data freshness**: Automatic 3-minute refresh cycles keep information current
5. **GitHub integration**: Direct access to documentation, code, and issue summaries

### Experience Targets
- **Voice-first**: Natural, low-friction conversation flow that prioritizes speech over screens
- **Audible feedback**: Cues for long-running or queued tasks maintain user awareness
- **Accessibility-first**: Ergonomic design with minimal UI distractions to keep attention on the task
- **Interruptibility**: Hardware/software interrupts propagate immediately through the system
